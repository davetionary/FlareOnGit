connectAP

connectAP is a Request and a Response message used by the client to connect to an available access point. The Request message contains data that defines which AP to connect to, and optional security data.

Conditions

  • noipv6 : A boolean indicating that IPv6 should be disabled on the connection (optional)
  • wep-index : A number representing the WEP key index
  • psk : A string representing the pre-shared encryption key
  • eap : A string representing the EAP type, one of "leap", "peap", "tls", or "ttls"
  • phase2-auth : The phase 2 authentication used with EAP, one of "mschapv2" or "gtc"
  • identity : A string representing the identity (e.g., user name) for EAP
  • password : A string representing the password used for EAP

Request (open AP):

{   "version" : 1,   "id" : 1,   "type" : "connectAP",   "data" : {       "ssid[%=Terminology.code-SSID%]" : "AnOpenAP"   }}

Request (Using WEP):

{   "version" : 1,   "id" : 1,   "type" : "connectAP",   "data" : {       "ssid[%=Terminology.code-SSID%]" : "NeverUseWEP",       "wep-key[%=Terminology.code-wep-key%]" : "AABBCCDDEEFF01020304050607",       "wep-index[%=Terminology.code-wep-key-index%]" : 1   }}

Request (Using pre-shared key encryption):

{   "version" : 1,   "id" : 1,   "type" : "connectAP",   "data" : {       "ssid[%=Terminology.code-SSID%]" : "NeedPassphrase",       "psk[%=Terminology.code-psk%]" : "my_secret_passphrase"   }}

Request (Using EAP):

{   "version" : 1,   "id" : 1,   "type" : "connectAP",   "data" : {       "ssid[%=Terminology.code-SSID%]" : "Supports8021X",       "eap[%=Terminology.code-eap%]" : "eap",       "phase2-auth[%=Terminology.code-phase2-auth%]" : "gtc"       "identity[%=Terminology.code-identity%]" : "username",       "password[%=Terminology.code-password%]" : "password"   }}

Response:

{   "version" : 1,   "id" : 1,   "type" : "connectAP",   "status" : 0}
